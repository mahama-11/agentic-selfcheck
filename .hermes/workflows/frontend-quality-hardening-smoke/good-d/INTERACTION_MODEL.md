# Interaction Model

## Primary user path

1. Open control room.
2. See overall status and urgent blockers.
3. Select a workstream.
4. Inspect next action and evidence.
5. Trigger or review follow-up action.

## Navigation / route choreography

Single-page prototype with left workstream rail, center operational board, right evidence/action inspector.

## Critical interactions

- Select workstream cards.
- Hover action buttons.
- Inspect evidence chips.
- Toggle density in v2.

## Mobile behavior

For demo, v1/v2 are desktop-first; real project would require mobile screenshot before final acceptance.
